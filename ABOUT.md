# About MarkItDown Workbench

## Suggested GitHub About

Local-first desktop workbench for batch document-to-Markdown conversion, review, cleanup, diagnostics, and export reports.

## Suggested topics

`markdown`, `document-conversion`, `local-first`, `desktop-app`, `batch-processing`, `developer-tools`, `productivity`, `tauri`, `typescript`

## Purpose

The project will provide a visible workflow around document conversion: users add files, review conversion state, inspect output, apply named cleanup profiles, and export Markdown together with a structured manifest.

## Intended users

- Researchers preparing readable document collections.
- Developers converting documentation and office files.
- Data teams preparing source material for later processing.
- Writers and editors who need consistent Markdown output.
- Privacy-conscious users who prefer local workflows.

## Scope

Queue management, local conversion adapters, preview, explicit cleanup rules, structured reports, and reproducible export.

## Non-goals

The project will not claim perfect layout reconstruction, automatic editorial correctness, or official affiliation with upstream libraries.

## Current status

Foundation phase. No executable release is available. The repository currently records product scope and release direction.

## Maintenance

Implemented behavior, screenshots, measurements, and compatibility badges will be added only after verification. Meaningful changes will be recorded through focused commits and release notes.
